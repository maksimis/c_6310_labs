# Лабораторная работа №4 — Решение студента №21

## Вариант
Номер варианта = `21 % 4 = 1` → **Вариант 1: Калькулятор с произвольным числом операндов**.

## Описание
Программа `calc` принимает имя арифметической операции и список целых чисел,
выполняет операцию и выводит результат.

```
./calc <операция> <число> [число...]
```

Поддерживаемые операции:

- `sum` — сумма всех чисел;
- `mul` — произведение всех чисел;
- `avg` — среднее арифметическое (целое, округление вниз,
  в том числе для отрицательных значений).

## Как это соотносится с требованиями

- **Указатели на функции** — массив `t_op ops[]` связывает имя операции с
  указателем на функцию `int (*)(int, ...)`. Выбор обработчика выполняется
  функцией `find_op` (`src/main.c`).
- **Вариативная функция (`...` + `va_list`)** — обработчики `op_sum`,
  `op_mul`, `op_avg` объявлены как `int op_*(int count, ...)` и читают
  операнды через `va_start` / `va_arg` / `va_end`.
- **Аргументы командной строки** — `main` разбирает `argc`/`argv`, проверяет
  валидность, парсит числа через `ft_atoi`.
- **libft** — весь вывод идёт через `ft_putstr_fd`, `ft_putnbr_fd`,
  `ft_putchar_fd`; для строк и сравнения используются `ft_strlen`,
  `ft_strncmp`; для парсинга — `ft_atoi`. Стандартные `stdio.h`, `stdlib.h`
  (кроме `malloc`/`free`), `string.h` не используются.
- **Обработка ошибок** — все сообщения идут в `stderr` через
  `ft_putstr_fd("...", 2)`, программа завершается с кодом `1`.

## Сборка

```
make          # собрать libft и ./calc
make clean    # удалить объектные файлы
make fclean   # удалить объектные файлы и бинарники
make re       # пересобрать
```

Компиляция выполняется с флагами `-Wall -Wextra -Werror`.

## Примеры

```
$ ./calc sum 1 2 3
6
$ ./calc mul 2 3 4
24
$ ./calc avg 10 20 30
20
$ ./calc sum -5 5 10
10
$ ./calc foo 1 2
Error: unknown operation: foo
$ ./calc sum
Usage: ./calc <sum|mul|avg> <num> [num...]
$ ./calc sum 1 abc
Error: invalid number: abc
```

## Ограничение

Из-за особенностей вызова функций с переменным числом аргументов в стандартном
C (невозможно forward-ить произвольное число аргументов в runtime) количество
операндов ограничено `MAX_NUMS = 32`. Для калькулятора из задания этого более
чем достаточно.

## Проверка утечек памяти

```
valgrind --leak-check=full --show-leak-kinds=all ./calc sum 1 2 3
```
Должно быть `All heap blocks were freed -- no leaks are possible`.
