matrix — лабораторная работа C (вариант 3)
Программа демонстрирует работу библиотеки matrix для матриц целых чисел.
В варианте 3 реализовано умножение двух матриц: mult_matrix(matrix_t *A, matrix_t *B).

Сборка
Требования:

компилятор C (cc или gcc)

библиотека libft в виде подкаталога ./libft

Makefile внутри libft, собирающий libft.a

Сборка:

bash
make
Будет собран исполняемый файл ./matrix.

Очистка объектных файлов:

bash
make clean
Полная очистка (включая бинарник):

bash
make fclean
Пересборка:

bash
make re
Формат запуска
bash
./matrix
Программа не принимает аргументы командной строки.
Все тестовые данные задаются внутри main.c.

Что делает программа
Программа:

создаёт матрицы через create_matrix;

заполняет их значениями в main.c;

выводит исходные матрицы через функции libft;

вычисляет произведение матриц через mult_matrix;

выводит результат;

освобождает всю выделенную память через remove_matrix.

Реализованные функции
matrix_t *create_matrix(int rows, int cols) — выделение памяти под матрицу;

void remove_matrix(matrix_t *A) — освобождение памяти;

void print_matrix(matrix_t *A) — вывод матрицы в stdout;

matrix_t *mult_matrix(matrix_t *A, matrix_t *B) — умножение двух матриц.

Пример
Если в main.c заданы матрицы:

text
A = 1  2  3
    4  5  6

B = 7   8
    9  10
   11  12
то результат работы программы:

text
Matrix A:
1 2 3
4 5 6
Matrix B:
7 8
9 10
11 12
A * B:
58 64
139 154
Обработка ошибок
Библиотека проверяет:

нулевые или отрицательные размеры матрицы;

ошибки выделения памяти;

NULL в указателях на матрицы;

несовместимые размеры матриц при умножении.

При ошибках:

функция create_matrix возвращает NULL;

функция mult_matrix возвращает NULL, если A->columns != B->rows;

сообщения об ошибках могут выводиться через функции libft;

программа завершает работу с ненулевым кодом возврата при критической ошибке.

Пример ошибки
Если попытаться умножить несовместимые матрицы, результат не будет создан:

text
Error: matrices cannot be multiplied
Проверка
Проект должен:

собираться с флагами -Wall -Wextra -Werror;

не иметь утечек памяти;

корректно освобождать все выделенные ресурсы;

использовать функции libft для вывода.
