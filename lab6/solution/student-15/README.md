# txtfile — лабораторная работа C (вариант 3)

Программа демонстрирует работу библиотеки `txtfile` для текстовых файлов.
В варианте 3 реализована замена подстроки во всём файле: `replace_substr(char **lines, const char *old, const char *new_str)`.

## Сборка

Требования:

- компилятор C (`cc` или `gcc`);
- библиотека `libft` в виде подкаталога `./libft`;
- `Makefile` внутри `libft`, собирающий `libft.a`.

Сборка:

```bash
make
```

Будет собран исполняемый файл `./replace_file`.

Очистка объектных файлов:

```bash
make clean
```

Полная очистка (включая бинарник):

```bash
make fclean
```

Пересборка:

```bash
make re
```

## Формат запуска

```bash
./replace_file <filename> <old> <new>
```

Пример запуска:

```bash
./replace_file data/input.txt cat fox
```

Где:

- `filename` — путь к существующему текстовому файлу;
- `old` — подстрока, которую нужно заменить;
- `new` — новая подстрока.

## Что делает программа

Программа:

- получает путь к файлу и параметры замены через `argv`;
- проверяет количество аргументов;
- считывает текстовый файл через `read_file`;
- выводит исходное содержимое файла;
- создаёт новый массив строк с заменой всех вхождений `old` на `new` через `replace_substr`;
- выводит результат замены;
- освобождает всю выделенную память через `free_lines`.

Исходный файл на диске не изменяется.
Замена выполняется в памяти, а результат выводится в `stdout`.

## Реализованные функции

- `char **read_file(const char *filename)` — считывает существующий текстовый файл и возвращает его содержимое в виде массива строк, заканчивающегося `NULL`;
- `void free_lines(char **lines)` — освобождает память, выделенную под массив строк;
- `void print_lines(char **lines)` — выводит массив строк в `stdout`;
- `char **replace_substr(char **lines, const char *old, const char *new_str)` — возвращает новый массив строк, где все вхождения `old` заменены на `new_str`.

## Пример

Если файл `data/input.txt` содержит:

```text
cat is here
black cat
dog and cat
```

то команда:

```bash
./replace_file data/input.txt cat fox
```

даст результат:

```text
Original file:
cat is here
black cat
dog and cat
Replaced file:
fox is here
black fox
dog and fox
```

## Обработка ошибок

Библиотека и программа проверяют:

- `NULL` в аргументах;
- отсутствие файла;
- невозможность открыть файл;
- ошибки чтения файла;
- ошибки выделения памяти;
- пустой файл;
- некорректное количество аргументов командной строки.

При ошибках:

- `read_file` возвращает `NULL`;
- `replace_substr` возвращает `NULL` при некорректных аргументах или ошибке выделения памяти;
- `main.c` выводит сообщение об ошибке через функции `libft` и завершает работу с ненулевым кодом возврата.

Пример ошибки:

```text
Usage: ./replace_file <filename> <old> <new>
```

или:

```text
Error: cannot read file
```

## Проверка

Проект должен:

- собираться с флагами `-Wall -Wextra -Werror`;
- не иметь утечек памяти;
- корректно освобождать все выделенные ресурсы;
- использовать `libft` для вывода;
- корректно работать с текстом построчно;
- не изменять исходный файл.

Пример проверки через `valgrind`:

```bash
valgrind --leak-check=full ./replace_file data/input.txt cat fox
```
