## Предупреждения -Wall

### 1. Неиспользуемый параметр функции (unused-parameter)
- **Строка:** 12
- **Код:** `void print_hello(int x) { printf("Hello\n"); }`
- **Текст предупреждения:** `warning: unused parameter 'x' [-Wunused-parameter]`
- **Объяснение:** Функция `print_hello` принимает параметр `x`, но внутри функции он не используется. Это может указывать на ошибку в логике или на незавершенную реализацию функции. Компилятор предупреждает о потенциально лишнем параметре.

### 2. Неиспользуемая переменная (unused-variable)
- **Строка:** 31
- **Код:** `int unused_counter = 0;`
- **Текст предупреждения:** `warning: unused variable 'unused_counter' [-Wunused-variable]`
- **Объяснение:** Переменная `unused_counter` объявлена и инициализирована, но ни разу не используется в программе. Такие переменные занимают память, но не приносят пользы и могут быть удалены.

### 3. Неиспользуемая статическая функция (unused-function)
- **Строка:** 17
- **Код:** `static void unused_helper(void) { printf("This function is never called\n"); }`
- **Текст предупреждения:** `warning: 'unused_helper' defined but not used [-Wunused-function]`
- **Объяснение:** Функция объявлена как `static` (видима только в этом файле), но нигде не вызывается. Это "мертвый" код, который можно удалить без влияния на работу программы.

### 4. Отсутствие return в функции (return-type)
- **Строка:** 24
- **Код:** `int get_number(void) { }`
- **Текст предупреждения:** `warning: control reaches end of non-void function [-Wreturn-type]`
- **Объяснение:** Функция объявлена как возвращающая `int`, но не содержит оператора `return`. При попытке использовать возвращаемое значение такой функции произойдет неопределенное поведение программы.

### 5. Сравнение знакового и беззнакового типа (sign-compare)
- **Строка:** 48
- **Код:** `if (size_val < n)`
- **Текст предупреждения:** `warning: comparison of integer expressions of different signedness: 'size_t' and 'int' [-Wsign-compare]`
- **Объяснение:** Переменная `size_val` имеет беззнаковый тип `size_t`, а `n` — знаковый `int`. Сравнение значений разных знаков может дать неожиданные результаты, особенно если знаковое число отрицательное, так как оно будет преобразовано в большое беззнаковое.

### 6. Использование неинициализированной переменной (uninitialized)
- **Строка:** 54
- **Код:** `printf("exact_uninit = %d\n", exact_uninit);`
- **Текст предупреждения:** `warning: 'exact_uninit' is used uninitialized in this function [-Wuninitialized]`
- **Объяснение:** Переменная `exact_uninit` объявлена, но ей не присвоено начальное значение. Использование такой переменной приводит к чтению мусорного значения из памяти, что является неопределенным поведением. Компилятор точно знает, что переменная не была инициализирована.

### 7. Отсутствие скобок в условии (parentheses)
- **Строка:** 58
- **Код:** `if (x = 5 || y == 10)`
- **Текст предупреждения:** `warning: suggest parentheses around assignment used as truth value [-Wparentheses]`
- **Объяснение:** В условии используется присваивание `x = 5` вместо сравнения `x == 5`. Это может быть ошибкой программиста. Компилятор рекомендует добавить скобки, чтобы явно указать намерение.

### 8. Использование char как индекса массива (char-subscripts)
- **Строка:** 65
- **Код:** `int val = array[index_char];`
- **Текст предупреждения:** `warning: array subscript has type 'char' [-Wchar-subscripts]`
- **Объяснение:** В качестве индекса массива используется переменная типа `char`. В зависимости от реализации `char` может быть знаковым или беззнаковым, и использование его как индекса может привести к отрицательным значениям индекса (если символ с кодом > 127), что вызовет выход за границы массива. Рекомендуется использовать `unsigned char` или явное приведение типа.

### 9. Пустое тело условного оператора (empty-body)
- **Строка:** 69
- **Код:** `if (n == 0); { printf("n равно нулю\n"); }`
- **Текст предупреждения:** `warning: suggest braces around empty body in an 'if' statement [-Wempty-body]`
- **Объяснение:** После условия `if (n == 0)` стоит точка с запятой, которая образует пустое тело оператора. Блок в фигурных скобках выполняется всегда, независимо от условия, что не соответствует замыслу программиста.

### 10. Деление на ноль (div-by-zero)
- **Строка:** 74
- **Код:** `int result = 10 / 0;`
- **Текст предупреждения:** `warning: division by zero [-Wdiv-by-zero]`
- **Объяснение:** Выполняется деление на ноль, что приводит к неопределенному поведению программы (обычно к аварийному завершению или некорректному результату). Компилятор обнаруживает эту операцию на этапе компиляции.

## Предупреждения -Wextra (бонус)

### 1. Сравнение с всегда истинным выражением (type-limits)
- **Строка:** 86
- **Код:** `unsigned int u_val = 5; if (u_val >= 0) { printf("u_val >= 0 всегда истинно\n"); }`
- **Текст предупреждения:** `warning: comparison of unsigned expression >= 0 is always true [-Wtype-limits]`
- **Объяснение:** Для беззнакового типа unsigned int условие >= 0 всегда истинно, так как беззнаковые числа не могут быть отрицательными. Эта проверка бессмысленна.

### 2. Переменная присваивается, но не используется (unused-but-set-variable)
- **Строка:** 92
- **Код:** `int temp_var = 10; temp_var = 20;`
- **Текст предупреждения:** `warning: variable 'temp_var' set but not used [-Wunused-but-set-variable]`
- **Объяснение:** Переменной temp_var присваиваются значения, но после этого она нигде не используется. Компилятор предупреждает о лишних операциях, которые можно удалить.

### 3. Переполнение при присваивании (overflow)
- **Строка:** 97
- **Код:** `char overflow_char = 300;`
- **Текст предупреждения:** `warning: overflow in conversion from 'int' to 'char' changes value from '300' to '44' [-Woverflow]`
- **Объяснение:** Значение 300 выходит за пределы диапазона типа char (обычно -128..127 или 0..255). Произойдет усечение значения, что приведет к потере данных и неожиданному результату.

### 4. Переполнение при сдвиге (shift-count-overflow)
- **Строка:** 101
- **Код:** `int shift_overflow = 1 << 40;`
- **Текст предупреждения:** `warning: left shift count >= width of type [-Wshift-count-overflow]`
- **Объяснение:** Попытка сдвинуть число на 40 бит, в то время как тип int на большинстве платформ занимает 32 бита. Результат такого сдвига не определен стандартом C и поведение программы становится непредсказуемым.

### 5. Возможно использование неинициализированной переменной (maybe-uninitialized)
- **Строка:** 106-108
- **Код:** `int maybe_uninit; if (n > 100) { maybe_uninit = 42; } printf("maybe_uninit = %d\n", maybe_uninit);`
- **Текст предупреждения:** `warning: 'maybe_uninit' may be used uninitialized in this function [-Wmaybe-uninitialized]`
- **Объяснение:** Переменная maybe_uninit инициализируется только в одной ветке условного оператора (когда n > 100). Если условие не выполнится (например, при вводе числа 5), переменная останется неинициализированной, и её использование приведет к чтению мусорного значения.
