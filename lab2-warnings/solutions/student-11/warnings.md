## Предупреждения -Wall (12 шт.)

### 1. Неиспользуемая переменная `x`
- **Строка:** 17
- **Код:** `int x = 10;`
- **Текст предупреждения:** `warning: unused variable 'x' [-Wunused-variable]`
- **Объяснение:** Переменная объявлена и инициализирована, но нигде не используется.

### 2. Неиспользуемая переменная `c`
- **Строка:** 35
- **Код:** `int c = b;`
- **Текст предупреждения:** `warning: unused variable 'c' [-Wunused-variable]`
- **Объяснение:** Переменная `c` получает значение, но далее не используется.

### 3. Сравнение знакового и беззнакового типа
- **Строка:** 57
- **Код:** `if (number < dummy_unsigned)`
- **Текст предупреждения:** `warning: comparison of integer expressions of different signedness: 'long int' and 'unsigned int' [-Wsign-compare]`
- **Объяснение:** Сравнение знакового `long` с беззнаковым `unsigned int` может дать неожиданный результат при отрицательных значениях.

### 4. Неиспользуемый результат выражения
- **Строка:** 78
- **Код:** `number + 10;`
- **Текст предупреждения:** `warning: statement with no effect [-Wunused-value]`
- **Объяснение:** Вычисляется сумма, но результат никуда не сохраняется.

### 5. Использование `char` как индекса массива
- **Строка:** 83
- **Код:** `dummy_array[idx] = 42;`
- **Текст предупреждения:** `warning: array subscript has type 'char' [-Wchar-subscripts]`
- **Объяснение:** Индекс массива имеет тип `char`, что может привести к ошибке при отрицательном значении. Рекомендуется использовать `int` или `size_t`.

## Предупреждения -Wextra (6 шт.)

### 1. Сравнение беззнакового выражения с нулём (всегда истинно)
- **Строка:** 69
- **Код:** `if (unsigned_number >= 0)`
- **Текст предупреждения:** `warning: comparison of unsigned expression in '>= 0' is always true [-Wtype-limits]`
- **Объяснение:** Для беззнакового типа условие `>= 0` всегда истинно, что может свидетельствовать о логической ошибке.

### 2. Сдвиг за пределы размера типа
- **Строка:** 89
- **Код:** `unsigned long shift_test = 1UL << 64;`
- **Текст предупреждения:** `warning: left shift count >= width of type [-Wshift-count-overflow]`
- **Объяснение:** Сдвиг на 64 бита при ширине типа `unsigned long` (обычно 64) приводит к неопределённому поведению.

### 3. Пропущены поля при инициализации структуры
- **Строка:** 97
- **Код:** `struct Point p = {10, 20};`
- **Текст предупреждения:** `warning: missing initializer for field 'z' of 'struct Point' [-Wmissing-field-initializers]`
- **Объяснение:** Структура содержит три поля, но инициализированы только два. Оставшееся поле будет обнулено, но предупреждение указывает на возможную оплошность.

### 4. Переменная `dummy_array` присвоена, но не использована
- **Строка:** 81
- **Код:** `int dummy_array[10] = {0};`
- **Текст предупреждения:** `warning: variable 'dummy_array' set but not used [-Wunused-but-set-variable]`
- **Объяснение:** Переменной присваивается значение (массив инициализируется), но после этого она нигде не читается.

### 5. Переменная `unused_but_set` присвоена, но не использована
- **Строка:** 47
- **Код:** `int unused_but_set;`
- **Текст предупреждения:** `warning: variable 'unused_but_set' set but not used [-Wunused-but-set-variable]`
- **Объяснение:** Переменной присваивается значение (из неинициализированной `uninitialized_val`), но после этого она нигде не читается.

### 6. Возможно неинициализированная переменная `uninitialized_val`
- **Строка:** 75
- **Код:** `unused_but_set = uninitialized_val;`
- **Текст предупреждения:** `warning: 'uninitialized_val' may be used uninitialized [-Wmaybe-uninitialized]`
- **Объяснение:** Переменная `uninitialized_val` не инициализирована, но её значение используется в присваивании. Компилятор предупреждает о возможном неопределённом поведении.
