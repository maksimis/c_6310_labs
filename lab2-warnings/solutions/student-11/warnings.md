## Предупреждения -Wall (10+)

### 1. Неиспользуемая статическая функция
- **Строка:** 14
- **Код:** `static void unused_helper(void) { ... }`
- **Текст предупреждения:** `warning: 'unused_helper' defined but not used [-Wunused-function]`
- **Объяснение:** Функция объявлена как static, но нигде не вызывается.

### 2. Неиспользуемая переменная
- **Строка:** 15
- **Код:** `int x = 10;`
- **Текст предупреждения:** `warning: unused variable 'x' [-Wunused-variable]`
- **Объяснение:** Переменная `x` объявлена и инициализирована, но не используется.

### 3. Функция без прототипа (устаревший стиль)
- **Строка:** 19-21
- **Код:** `int old_style_function(); int old_style_function(p) int p; { return p * 2; }`
- **Текст предупреждения:** `warning: a function definition without a prototype is deprecated in all versions of C and is not supported in C23 [-Wdeprecated-non-prototype]`
- **Объяснение:** В современном C требуется указывать типы параметров в круглых скобках.

### 4. Неиспользуемая переменная `c`
- **Строка:** 33
- **Код:** `int c = b;`
- **Текст предупреждения:** `warning: unused variable 'c' [-Wunused-variable]`
- **Объяснение:** Переменная `c` объявлена, инициализирована, но нигде не используется.

### 5. Отсутствие возвращаемого значения в non-void функции
- **Строка:** 40
- **Код:** `int missing_return(int x) { x = x * 2; }`
- **Текст предупреждения:** `warning: control reaches end of non-void function [-Wreturn-type]`
- **Объяснение:** Функция объявлена как возвращающая `int`, но не содержит оператора `return`.

### 6. Неиспользуемые параметры `argc` и `argv`
- **Строка:** 43
- **Код:** `int main(int argc, char *argv[])`
- **Текст предупреждения:** `warning: unused parameter 'argc' [-Wunused-parameter]` и аналогично для `argv`
- **Объяснение:** Параметры командной строки объявлены, но не используются в программе.

### 7. Сравнение знакового и беззнакового типа
- **Строка:** 58
- **Код:** `if (number < dummy_unsigned)`
- **Текст предупреждения:** `warning: comparison of integer expressions of different signedness: 'long int' and 'unsigned int' [-Wsign-compare]`
- **Объяснение:** `number` — знаковый `long`, `dummy_unsigned` — беззнаковый. Сравнение может дать неожиданный результат при отрицательных значениях.

### 8. Пустое тело условного оператора
- **Строка:** 60
- **Код:** `;` (пустой оператор после `if`)
- **Текст предупреждения:** `warning: empty body in an if-statement [-Wempty-body]`
- **Объяснение:** После `if` стоит точка с запятой, тело отсутствует. Часто является ошибкой.

### 9. Использование неинициализированной переменной
- **Строка:** 78
- **Код:** `unused_but_set = uninitialized_val;`
- **Текст предупреждения:** `warning: 'uninitialized_val' is used uninitialized [-Wuninitialized]`
- **Объяснение:** Переменная `uninitialized_val` не получила начального значения, но её значение присваивается другой переменной.

### 10. Неиспользуемый результат выражения
- **Строка:** 82
- **Код:** `number + 10;`
- **Текст предупреждения:** `warning: statement with no effect [-Wunused-value]`
- **Объяснение:** Вычисляется сумма, но результат никуда не сохраняется и не используется.

### 11. Использование `char` как индекса массива
- **Строка:** 87
- **Код:** `dummy_array[idx] = 42;`
- **Текст предупреждения:** `warning: array subscript has type 'char' [-Wchar-subscripts]`
- **Объяснение:** Индекс массива имеет тип `char`, что может привести к ошибке, если символ отрицательный. Рекомендуется использовать `int` или `size_t`.

## Предупреждения -Wextra (бонус, 6 шт.)

### 1. Переменной присваивается значение, но не используется (unused-but-set)
- **Строка:** 44
- **Код:** `int unused_but_set;`
- **Текст предупреждения:** `warning: variable 'unused_but_set' set but not used [-Wunused-but-set-variable]`
- **Объяснение:** Переменной присваивается значение (из неинициализированной), но после этого она нигде не читается.

### 2. Сравнение беззнакового выражения с нулём (всегда истинно)
- **Строка:** 72
- **Код:** `if (unsigned_number >= 0)`
- **Текст предупреждения:** `warning: comparison of unsigned expression in '>= 0' is always true [-Wtype-limits]`
- **Объяснение:** Для беззнакового типа условие `>= 0` всегда выполняется, что может свидетельствовать о логической ошибке.

### 3. Переполнение при сдвиге
- **Строка:** 95
- **Код:** `unsigned long shift_test = 1UL << 64;`
- **Текст предупреждения:** `warning: left shift count >= width of type [-Wshift-count-overflow]`
- **Объяснение:** Сдвиг на 64 бита при ширине типа `unsigned long` (обычно 64) приводит к неопределённому поведению.

### 4. Пропущены поля при инициализации структуры
- **Строка:** 103
- **Код:** `struct Point p = {10, 20};`
- **Текст предупреждения:** `warning: missing initializer for field 'z' of 'struct Point' [-Wmissing-field-initializers]`
- **Объяснение:** Структура содержит три поля, но инициализированы только два. Оставшееся поле будет инициализировано нулём, но предупреждение указывает на возможную оплошность.

### 5. Возможно неинициализированная переменная
- **Строка:** 45
- **Код:** `int maybe_unused;` и затем `printf("Dummy: %d\n", maybe_unused);` (строка 115)
- **Текст предупреждения:** `warning: 'maybe_unused' may be used uninitialized [-Wmaybe-uninitialized]`
- **Объяснение:** Переменная объявлена, но не инициализирована, после чего её значение используется в `printf`. Компилятор не может гарантировать, что она будет инициализирована к моменту использования.

### 6. Параметр, которому присваивается значение, но не используется далее
- **Строка:** 30-32
- **Код:** `static void unused_param_example(int a, int b) { a = a + 1; ... }`
- **Текст предупреждения:** `warning: parameter 'a' set but not used [-Wunused-but-set-parameter]` (появляется при `-Wextra`)
- **Объяснение:** Параметру `a` присваивается новое значение, но это значение нигде не используется.
