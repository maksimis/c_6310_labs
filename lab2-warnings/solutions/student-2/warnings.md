# Отчет о предупреждениях
Студент: 2
Задание: 3 (Сортировка массива по возрастанию)

## Предупреждения -Wall

### 1. Неиспользуемая переменная
- Строка: 18
- Код: `int unused_variable;`
- Текст предупреждения: `warning: unused variable 'unused_variable' [-Wunused-variable]`
- Объяснение: Переменная объявлена, но нигде не используется. Компилятор предупреждает о потенциально лишнем коде.

### 2. Неиспользуемый параметр функции
- Строка: 50
- Код: `void print_array(int arr[], int arr_size, int unused_parameter) { ... }`
- Текст предупреждения: `warning: unused parameter 'unused_parameter' [-Wunused-parameter]`
- Объяснение: Параметр функции передан, но внутри функции не используется. Это может быть ошибкой или недосмотром.

### 3. Сравнение знакового и беззнакового
- Строка: 83
- Код: `if (a > b)` (где `a` – unsigned int, `b` – int)
- Текст предупреждения: `warning: comparison of integer expressions of different signedness: 'unsigned int' and 'int' [-Wsign-compare]`
- Объяснение: Сравнение беззнакового и знакового типов может привести к неожиданным результатам из-за правил неявного преобразования (знаковое значение превращается в беззнаковое).

### 4. Отсутствие скобок в сложном выражении
- Строка: 98
- Код: `if (a & b > c)`
- Текст предупреждения: `warning: suggest parentheses around comparison in operand of '&' [-Wparentheses]`
- Объяснение: Приоритет операторов `>` выше, чем у `&`, поэтому выражение вычисляется как `a & (b > c)`. Возможно, программист имел в виду `(a & b) > c`. Рекомендуется явно указывать скобки.

### 5. Вводящий в заблуждение отступ
- Строка: 105–107
- Код: 
  ```c
  if (x > 5)
      printf("One line\n");
      printf("This line is not part of the if...\n");
- Текст предупреждения: warning: this 'if' clause does not guard... [-Wmisleading-indentation]
- Объяснение: Из-за отсутствия фигурных скобок только первый printf относится к условию, хотя второй по отступу выглядит так, будто он тоже входит в тело if. Это может ввести в заблуждение читателя.

### 6. Пустое тело условного оператора
- Строка: 108
- Код: `if (x < 15);`
- Текст предупреждения: warning: suggest braces around empty body in an 'if' statement [-Wempty-body]
- Объяснение: После условия сразу стоит точка с запятой, тело оператора пустое. Часто это опечатка (например, лишняя ;).

### 7. Деление на ноль
- Строка: 112
- Код: `int result = 100 / 0;`
- Текст предупреждения: warning: division by zero [-Wdiv-by-zero]
- Объяснение: Деление на ноль приводит к неопределённому поведению. Даже если код находится в невыполняемой ветке, компилятор выдаёт предупреждение.

### 8. Отсутствие return в функции, которая должна возвращать значение
- Строка: 78
- Код: `int unreturned_function() { }`
- Текст предупреждения: warning: control reaches end of non-void function [-Wreturn-type]
- Объяснение: Функция объявлена как возвращающая int, но не содержит оператора return. Это приводит к неопределённому поведению при использовании возвращаемого значения.

### 9. Неиспользуемая статическая функция
- Строка: 72
- Код: `static void unused_function() { }`
- Текст предупреждения: warning: 'unused_function' defined but not used [-Wunused-function]
- Объяснение: Статическая функция определена, но нигде не вызывается. Это может быть лишним кодом.

### 10. Использование неинициализированной переменной
- Строка: 90
- Код: 
`int uninitialized_var;`
`int result = uninitialized_var + 10;`
- Текст предупреждения: warning: 'uninitialized_var' is used uninitialized [-Wuninitialized]
- Объяснение: Объяснение: Переменная не инициализирована, её значение содержит «мусор». Использование такого значения ведёт к непредсказуемому поведению.

## Предупреждения -Wextra (бонус)

### 1. сравнение, которое всегда истинно или ложно из-за диапазона типа
- Строка: 118
- Код: `if (unsigned_var >= 0) { ... }`
- Текст предупреждения: warning: comparison of unsigned expression in '>= 0' is always true [-Wtype-limits]
- Объяснение: Для беззнакового типа условие >= 0 всегда истинно, поэтому проверка избыточна и может скрывать ошибку проектирования.

### 2. Пропущены поля при инициализации структуры
- Строка:130
- Код:
`struct Point { int x; int y; int z; };`
`struct Point p = {1, 2};`
- Текст предупреждения: warning: missing initializer for field 'z' of 'struct Point' [-Wmissing-field-initializers]
- Объяснение: При инициализации структуры не указано значение для поля z. Хотя оно будет обнулено, предупреждение напоминает, что это могло быть сделано намеренно или случайно.

### 3. Переменной присваивается значение, но оно не используется
- Строка: 135
- Код: `int unused_but_set = 42;`
- Текст предупреждения: warning: variable 'unused_but_set' set but not used [-Wunused-but-set-variable]
- Объяснение: Переменная инициализирована, но нигде не читается. Это может быть лишним действием или признаком незавершённого кода.

### 4. Параметру присваивается значение, но оно не используется
- Строка: 140
- Код:
`void unused_but_set_parameter(int unused_parameter) {`
`    unused_parameter = 50;`
`}`
- Текст предупреждения: warning: parameter 'unused_parameter' set but not used [-Wunused-but-set-parameter]
- Объяснение: Параметру присваивается значение, но после этого он нигде не используется. Это может быть бессмысленной операцией.

### 5. Переполнение при константных вычислениях
- Строка: 145
- Код: `int result = INT_MAX + 1;`
- Текст предупреждения: warning: integer overflow in expression of type 'int' results in '-2147483648' [-Woverflow]
- Объяснение:  Сложение двух констант приводит к переполнению знакового целого. Поведение переполнения знаковых типов не определено стандартом (implementation-defined или undefined). Код, вероятно, не должен выполняться, но предупреждение полезно.
