# Отчет о предупреждениях
Студент: 4  
Задание: 5 (N-е число Фибоначчи)

## Предупреждения -Wall

### 1. Неиспользуемая статическая функция (unused-function)
- Строка: 13
- Код: `static void unused_func(void) { }`
- Текст предупреждения: `warning: 'unused_func' defined but not used [-Wunused-function]`
- Объяснение: Функция объявлена как `static`, но нигде не вызывается. Компилятор считает её определение излишним.

### 2. Функция без возвращаемого значения (return-type)
- Строка: 20
- Код: `int no_return_func(void) { }`
- Текст предупреждения: `warning: control reaches end of non-void function [-Wreturn-type]`
- Объяснение: Функция должна возвращать значение типа `int`, однако в ней отсутствует оператор `return`. Попытка использовать результат такой функции приведёт к неопределённому поведению.

### 3. Неиспользуемое значение (unused-value)
- Строка: 24
- Код: `1;`
- Текст предупреждения: `warning: statement with no effect [-Wunused-value]`
- Объяснение: Выражение `1;` вычисляется, но его результат никак не используется и не имеет побочных эффектов. Такие операторы не несут смысловой нагрузки.

### 4. Вводящий в заблуждение отступ (misleading-indentation)
- Строка: 31
- Код: `if (x == 0);`
- Текст предупреждения: `warning: this 'if' clause does not guard... [-Wmisleading-indentation]`
- Объяснение: После условия `if` стоит точка с запятой, поэтому тело условия пустое. Следующий блок (строки 32–34) выполняется всегда, но из-за отступа кажется, что он относится к `if`. Это может ввести в заблуждение читающего код.

### 5. Отсутствие скобок в логическом выражении (parentheses)
- Строка: 40
- Код: `if (a && b || c)`
- Текст предупреждения: `warning: suggest parentheses around '&&' within '||' [-Wparentheses]`
- Объяснение: В выражении смешаны операторы `&&` и `||` без скобок. Хотя приоритет операторов чётко определён, компилятор рекомендует явно указать скобки для улучшения читаемости и предотвращения возможных ошибок.

### 6. Использование char в качестве индекса массива (char-subscripts)
- Строка: 49
- Код: `int val = arr[idx];`
- Текст предупреждения: `warning: array subscript has type 'char' [-Wchar-subscripts]`
- Объяснение: Индекс массива имеет тип `char`. В зависимости от реализации `char` может быть знаковым, и отрицательное значение (например, `idx = -1`) приведёт к выходу за границы массива.

### 7. Несоответствие указателей по знаковости (pointer-sign)
- Строка: 56
- Код: `unsigned char *ucp = &c;`
- Текст предупреждения: `warning: pointer targets in initialization of 'unsigned char *' from 'char *' differ in signedness [-Wpointer-sign]`
- Объяснение: Инициализация указателя на `unsigned char` адресом переменной типа `char`. Указатели на знаковый и беззнаковый типы несовместимы по знаковости, что может вызвать проблемы при разыменовании.

### 8. Деление на ноль (div-by-zero)
- Строка: 62
- Код: `int x = 1 / 0;`
- Текст предупреждения: `warning: division by zero [-Wdiv-by-zero]`
- Объяснение: Попытка деления константы на ноль обнаруживается на этапе компиляции. Такая операция приводит к неопределённому поведению.

### 9. Использование неинициализированной переменной (uninitialized)
- Строка: 69
- Код: `int w = z;`
- Текст предупреждения: `warning: 'z' is used uninitialized [-Wuninitialized]`
- Объяснение: Переменная `z` не инициализирована, но её значение используется для инициализации `w`. Значение `z` не определено, что ведёт к непредсказуемому поведению программы.

### 10. Неиспользуемая переменная (unused-variable)
- Строка: 125
- Код: `int not_used;`
- Текст предупреждения: `warning: unused variable 'not_used' [-Wunused-variable]`
- Объяснение: Переменная объявлена, но нигде в программе не используется. Такие переменные обычно являются лишними и загромождают код.

## Предупреждения -Wextra (бонус)

### 1. Сравнение, которое всегда ложно (type-limits)
- Строка: 78
- Код: `if (u < 0)`
- Текст предупреждения: `warning: comparison of unsigned expression in '< 0' is always false [-Wtype-limits]`
- Объяснение: Переменная `u` имеет беззнаковый тип, поэтому условие `u < 0` всегда ложно. Такое сравнение бессмысленно и может указывать на ошибку.

### 2. Переменная присваивается, но не используется (unused-but-set-variable)
- Строка: 85
- Код: `int x = 5; x = 10;`
- Текст предупреждения: `warning: variable 'x' set but not used [-Wunused-but-set-variable]`
- Объяснение: Переменной `x` присваивается значение, но оно нигде не используется. Это может быть признаком лишнего кода или ошибки.

### 3. Параметру присваивается, но не используется (unused-but-set-parameter)
- Строка: 90
- Код: `void unused_but_set_param(int x) { x = 100; }`
- Текст предупреждения: `warning: parameter 'x' set but not used [-Wunused-but-set-parameter]`
- Объяснение: Параметру функции присваивается новое значение, но оно не используется. Обычно параметры служат для передачи данных, их изменение без использования может быть ошибкой.

### 4. Сдвиг на количество бит, превышающее размер типа (shift-count-overflow)
- Строка: 96
- Код: `int x = 1 << 40;`
- Текст предупреждения: `warning: left shift count >= width of type [-Wshift-count-overflow]`
- Объяснение: Сдвиг влево на 40 бит для типа `int` (обычно 32 бита) приводит к неопределённому поведению, так как количество бит сдвига превышает размер типа.

### 5. Пропущены поля при инициализации структуры (missing-field-initializers)
- Строка: 108
- Код: `struct S s = {1, 2};`
- Текст предупреждения: `warning: missing initializer for field 'c' of 'struct S' [-Wmissing-field-initializers]`
- Объяснение: При инициализации структуры указаны значения только для первых двух полей, а третье поле остаётся неинициализированным. Это может привести к непредсказуемым значениям.
