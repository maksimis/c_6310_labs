## Предупреждения -Wall (10 типов)

### 1. Неиспользуемая переменная — `unused_counter`
- **Строка:** 74
- **Код:** `int unused_counter = 0;`
- **Текст предупреждения:** `warning: unused variable 'unused_counter' [-Wunused-variable]`
- **Объяснение:** Переменная объявлена и инициализирована, но нигде не используется. Компилятор сообщает, что резервирование памяти и инициализация бесполезны.

### 2. Неиспользуемая переменная — `status_buffer`
- **Строка:** 78
- **Код:** `char status_buffer[32];`
- **Текст предупреждения:** `warning: unused variable 'status_buffer' [-Wunused-variable]`
- **Объяснение:** Массив объявлен, но ни разу не задействован. Компилятор предупреждает о бесполезном выделении памяти на стеке.

### 3. Неиспользуемая статическая функция
- **Строка:** 15
- **Код:** `static void debug_print(const char *message) { ... }`
- **Текст предупреждения:** `warning: 'debug_print' defined but not used [-Wunused-function]`
- **Объяснение:** Статическая функция видна только в пределах файла. Если она нигде не вызывается, компилятор считает её мёртвым кодом (dead code).

### 4. Отсутствие return в не-void функции
- **Строка:** 23
- **Код:** `int get_version(void) { /* Нет return */ }`
- **Текст предупреждения:** `warning: control reaches end of non-void function [-Wreturn-type]`
- **Объяснение:** Функция объявлена как возвращающая `int`, но не содержит оператора `return`. Поток выполнения достигает конца функции без возврата значения — неопределённое поведение при вызове.

### 5. Несовпадение знаковости указателей
- **Строка:** 52
- **Код:** `unsigned char *uptr = char_buf;`
- **Текст предупреждения:** `warning: pointer targets in initialization of 'unsigned char *' from 'char *' differ in signedness [-Wpointer-sign]`
- **Объяснение:** Переменная `char_buf` имеет тип `char *` (знаковый указатель), а `uptr` — `unsigned char *`. Компилятор предупреждает о потенциальных проблемах при разыменовании: знаковость влияет на интерпретацию байт.

### 6. Отсутствие скобок в составном выражении
- **Строка:** 113
- **Код:** `int combined = first | second & SIGN_MASK;`
- **Текст предупреждения:** `warning: suggest parentheses around arithmetic in operand of '|' [-Wparentheses]`
- **Объяснение:** Оператор `&` имеет больший приоритет, чем `|`, поэтому выражение вычисляется как `first | (second & SIGN_MASK)`. Компилятор рекомендует добавить скобки, чтобы намерение было явным.

### 7. Переменная типа char как индекс массива
- **Строка:** 120
- **Код:** `char selected = hex_digits[hex_index];`
- **Текст предупреждения:** `warning: array subscript has type 'char' [-Wchar-subscripts]`
- **Объяснение:** Тип `char` может быть знаковым (реализация-зависимо). Если значение `char` отрицательно, индекс массива станет отрицательным, что вызовет выход за границы. Безопаснее использовать `int` или `unsigned char` в качестве индекса.

### 8. Результат выражения не используется
- **Строка:** 125
- **Код:** `first + second;`
- **Текст предупреждения:** `warning: statement with no effect [-Wunused-value]`
- **Объяснение:** Результат сложения двух переменных нигде не сохраняется и не используется. Вычисление производится впустую. Компилятор предупреждает о бесполезном операторе-выражении.

### 9. Пропущены скобки в инициализаторе
- **Строка:** 92
- **Код:** `int matrix[2][2] = {1, 2, 3, 4};`
- **Текст предупреждения:** `warning: missing braces around initializer [-Wmissing-braces]`
- **Объяснение:** При инициализации многомерного массива рекомендуется явно выделять строки: `{{1, 2}, {3, 4}}`. Без внутренних скобок компилятор предупреждает о неявном порядке заполнения элементов.

### 10. Переполнение при присваивании константы
- **Строка:** 149
- **Код:** `unsigned char byte_overflow = 300;`
- **Текст предупреждения:** `warning: unsigned conversion from 'int' to 'unsigned char' changes value from '300' to '44' [-Woverflow]`
- **Объяснение:** Тип `unsigned char` хранит значения от 0 до 255. Константа 300 не вмещается: при усечении до 8 бит получается 300 % 256 = 44. Данные теряются.

### 11. Вводящий в заблуждение отступ
- **Строки:** 155–157
- **Код:**
  ```c
  if (result > 0)
      printf("НОК положительный.\n");
      log_output(result, 0);
  ```
- **Текст предупреждения:** `warning: this 'if' clause does not guard... [-Wmisleading-indentation]`
- **Объяснение:** Оператор `log_output` отступом напоминает продолжение тела `if`, но из-за отсутствия фигурных скобок он выполняется безусловно. Компилятор предупреждает, что отступ вводит читателя в заблуждение.

## Предупреждения только -Wextra (бонус, 7 типов)

### 1. Неиспользуемый параметр функции
- **Строка:** 27
- **Код:** `static void print_result(int first, int second, int result, int precision)`
- **Текст предупреждения:** `warning: unused parameter 'precision' [-Wunused-parameter]`
- **Объяснение:** Параметр `precision` объявлен, но нигде в теле функции не используется. `-Wextra` расширяет проверки неиспользуемых сущностей до параметров функций.

### 2. Сравнение знакового и беззнакового типов
- **Строка:** 35
- **Код:** `if (value < max_range)`
- **Текст предупреждения:** `warning: comparison of integer expressions of different signedness: 'int' and 'unsigned int' [-Wsign-compare]`
- **Объяснение:** `value` имеет тип `int` (может быть отрицательным), `max_range` — `unsigned int`. При сравнении отрицательный `int` неявно преобразуется в большое беззнаковое число, что меняет смысл сравнения.

### 3. Параметру присваивается значение, которое не читается
- **Строка:** 43
- **Код:** `base = 10;`
- **Текст предупреждения:** `warning: parameter 'base' set but not used [-Wunused-but-set-parameter]`
- **Объяснение:** Параметру `base` присваивается значение 10 внутри функции, однако это новое значение нигде не читается. Первоначальное (переданное) значение также игнорируется. Присваивание бессмысленно.

### 4. Неполная инициализация структуры
- **Строка:** 87
- **Код:** `struct coord origin = {0, 0};`
- **Текст предупреждения:** `warning: missing initializer for field 'z' of 'struct coord' [-Wmissing-field-initializers]`
- **Объяснение:** Структура `coord` содержит три поля (`x`, `y`, `z`), но инициализатор явно задаёт только два. Поле `z` будет обнулено неявно, однако компилятор предупреждает о потенциально случайном пропуске.

### 5. Пустое тело оператора if
- **Строка:** 109
- **Код:** `if (debug_flag);`
- **Текст предупреждения:** `warning: suggest braces around empty body in an 'if' statement [-Wempty-body]`
- **Объяснение:** Тело оператора `if` — пустой оператор (точка с запятой). Это может быть случайной опечаткой (лишняя `;`). Компилятор рекомендует заменить `;` на явные `{}`.

### 6. Сравнение беззнакового типа с нулём (всегда истинно)
- **Строка:** 143
- **Код:** `if (check_val >= 0)`
- **Текст предупреждения:** `warning: comparison of unsigned expression in '>= 0' is always true [-Wtype-limits]`
- **Объяснение:** `check_val` имеет тип `unsigned int`, который не может быть отрицательным по определению. Условие `>= 0` всегда истинно, и проверка бессмысленна.

### 7. Неявный провал в следующий case
- **Строка:** 132
- **Код:** `case 0: printf("Первое число кратно 3.\n");`
- **Текст предупреждения:** `warning: this statement may fall through [-Wimplicit-fallthrough=]`
- **Объяснение:** В ветке `case 0` отсутствует оператор `break`, поэтому выполнение «проваливается» в `case 1`. Компилятор предупреждает, что это, вероятно, ошибка. Для намеренного провала следует добавить комментарий `/* falls through */`.
