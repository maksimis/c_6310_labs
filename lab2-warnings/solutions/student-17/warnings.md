# Отчет о предупреждениях
**Студент:** 17  
**Задание:** 18. Сумма первых n натуральных чисел  
**Компилятор:** GCC  
**Флаг:** `-Wall -Wextra`

---

## Предупреждения -Wall

### 1. Неявное определение типа возвращаемого значения
- **Строка:** 18
- **Код:** `oldStyleFunction(a, b)`
- **Текст ошибки:** `error: return type defaults to 'int' [-Wimplicit-int]`
- **Объяснение:** В определении функции не указан тип возвращаемого значения. В старых версиях C это по умолчанию означало `int`, но в современном C это считается ошибкой. Необходимо явно указывать тип возвращаемого значения.

---

### 2. Функция в старом стиле (без прототипа)
- **Строка:** 18
- **Код:** `oldStyleFunction(a, b)`
- **Текст предупреждения:** `warning: old-style function definition [-Wold-style-definition]`
- **Объяснение:** Функция определена в устаревшем стиле K&R, где типы параметров указываются после списка параметров. В современном C следует использовать прототипы функций с указанием типов параметров: `int oldStyleFunction(int a, int b)`.

---

### 3. Неиспользуемая переменная
- **Строка:** 26
- **Код:** `int x = 10;`
- **Текст предупреждения:** `warning: unused variable 'x' [-Wunused-variable]`
- **Объяснение:** Переменная `x` объявлена и инициализирована, но нигде не используется в функции `noReturnValue()`. Это может указывать на логическую ошибку или лишний код.

---

### 4. Присваивание в условии (lvalue required)
- **Строка:** 44
- **Код:** `if (scanf("%d", &number) = 1)`
- **Текст ошибки:** `error: lvalue required as left operand of assignment`
- **Объяснение:** В условии `if` используется оператор присваивания `=` вместо оператора сравнения `==`. Результат вызова функции `scanf()` нельзя изменить, поэтому его нельзя использовать в левой части присваивания. Правильно: `if (scanf("%d", &number) == 1)`.

---

### 5. Бессмысленное сравнение (сравнение с самим собой)
- **Строка:** 50
- **Код:** `if (number == number)`
- **Текст предупреждения:** `warning: self-comparison always evaluates to true [-Wtautological-compare]`
- **Объяснение:** Сравнение переменной с самой собой всегда будет истинным. Это бессмысленное условие, которое не выполняет никакой полезной проверки и, вероятно, является ошибкой.

---

### 6. Вводящее в заблуждение форматирование (misleading indentation)
- **Строка:** 55–56
- **Код:** 
  ```c
  if (number <= 0);
  {
      printf("Ошибка! Нужно ввести натуральное число.\n");
      return 1;
  }
  ```
- **Текст предупреждения:** `warning: this 'if' clause does not guard... [-Wmisleading-indentation]`
- **Объяснение:** После `if` стоит точка с запятой `;`, которая завершает тело условного оператора. Блок кода в фигурных скобках выполняется всегда, независимо от условия, но из-за форматирования создается ложное впечатление, что он принадлежит `if`. Это классическая ошибка, приводящая к логическим багам.

---

### 7. Присваивание в условии цикла (без скобок)
- **Строка:** 62
- **Код:** `for (int num = 1; num = number; num++)`
- **Текст предупреждения:** `warning: suggest parentheses around assignment used as truth value [-Wparentheses]`
- **Объяснение:** В условии цикла используется присваивание `=` вместо сравнения `<=`. Хотя это синтаксически допустимо (присваивание возвращает присвоенное значение), это почти всегда является ошибкой. Если это действительно задумано, следует добавить дополнительные скобки: `for (int num = 1; (num = number); num++)`.

---

### 8. Оператор без эффекта
- **Строка:** 68
- **Код:** `uninitializedVar;`
- **Текст предупреждения:** `warning: statement with no effect [-Wunused-value]`
- **Объяснение:** Строка содержит только имя переменной без какой-либо операции. Это не имеет никакого эффекта во время выполнения и, вероятно, является ошибкой. Возможно, предполагалось использовать значение переменной или присвоить ей что-то.

---

### 9. Неявное объявление функции
- **Строка:** 71
- **Код:** `implicitFunctionCall();`
- **Текст ошибки:** `error: implicit declaration of function 'implicitFunctionCall' [-Wimplicit-function-declaration]`
- **Объяснение:** Функция `implicitFunctionCall()` вызывается до того, как она объявлена или определена. В старых стандартах C это допускалось с неявным объявлением, но в современном C это ошибка. Необходимо либо объявить прототип функции перед вызовом, либо переместить определение функции выше.

---

### 10. Неправильный спецификатор формата
- **Строка:** 74
- **Код:** `printf("Число: %s\n", number);`
- **Текст предупреждения:** `warning: format '%s' expects argument of type 'char *', but argument 2 has type 'int' [-Wformat=]`
- **Объяснение:** Спецификатор формата `%s` требует аргумент типа `char*` (строку), но передается `int` (число). Это приводит к неопределенному поведению при выполнении. Правильно использовать `%d` для целых чисел.

---

### 11. Неиспользуемая переменная
- **Строка:** 34
- **Код:** `int unusedVariable;`
- **Текст предупреждения:** `warning: unused variable 'unusedVariable' [-Wunused-variable]`
- **Объяснение:** Переменная `unusedVariable` объявлена, но никогда не используется в программе. Такие переменные занимают память и загромождают код без необходимости.

---

### 12. Неиспользуемая переменная
- **Строка:** 40
- **Код:** `unsigned int unsignedNum = 5;`
- **Текст предупреждения:** `warning: unused variable 'unsignedNum' [-Wunused-variable]`
- **Объяснение:** Переменная `unsignedNum` объявлена и инициализирована, но не используется. Возможно, она была предназначена для демонстрации сравнения знаковых и беззнаковых типов, но так и не была применена.

---

### 13. Конфликтующие типы функции
- **Строка:** 81
- **Код:** `void implicitFunctionCall()`
- **Текст предупреждения:** `warning: conflicting types for 'implicitFunctionCall'; have 'void(void)'`
- **Объяснение:** Тип функции при определении конфликтует с неявным объявлением, которое было создано при первом вызове. Хотя в данном случае типы совпадают, компилятор предупреждает о потенциальных проблемах. Лучше явно объявлять все функции перед использованием.

---

### 14. Отсутствие возвращаемого значения
- **Строка:** 28
- **Код:** `}`
- **Текст предупреждения:** `warning: control reaches end of non-void function [-Wreturn-type]`
- **Объяснение:** Функция `noReturnValue()` объявлена как возвращающая `int`, но не содержит оператора `return`. В C это приводит к неопределенному поведению, так как вызывающий код ожидает получить значение, но функция его не возвращает.

---

## Предупреждения -Wextra (бонус)

При компиляции с флагом `-Wextra` дополнительных предупреждений не обнаружено. Все перечисленные выше предупреждения относятся к категориям, уже покрываемым флагом `-Wall`.
